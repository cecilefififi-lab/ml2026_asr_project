## 2. 本次实验在做什么

实验三回答的问题是：当 ASR 已经把语音识别成错误文本后，LLM 作为“文本校对员”还能补救多少？它更擅长修近音字、术语错，还是更适合发现整段幻觉并拒绝脑补？

本次实验选了 5 条代表性错误，覆盖四类情况：

| 案例 | 来源 | 错误类型 | 原始 ASR 输出 | 参考文本 |
|---|---|---|---|---|
| case1 | `con_005 babble_5dB whisper` | 术语错，上下文基本完整 | 我觉得晚睡晚起是好意 | 哦觉得晚睡晚起是熬夜 |
| case2 | `pro_005 clean whisper` | 术语错，但声学差得较远 | 好 我先聊玩具 | 好 我先聊晚睡晚起 |
| case3 | `pro_015 white_0dB funasr+frcrn` | 近音错，两句话都通顺 | 我说了两个小时 | 我多了两个小时 |
| case4 | `con_009 white_0dB whisper` | 综艺名幻觉 | 小明星大跟班下次再見 | 早睡晚睡 |
| case5 | `pro_002 white_0dB whisper+specsub` | 字幕式幻觉 | 請不吝点赞 订阅 转发 打赏支持明镜与点点栏目 | 没问题没问题先不着急 |

实验采用盲测方式：纠错时只给 ASR 输出文本，不给参考答案。这样可以避免 LLM 通过答案本身“作弊”，更接近真实后处理场景。

对比了三种设置：

```text
1. 无纠错：直接保留 ASR 原始输出
2. 纯 LLM：只把 ASR 文本交给 LLM 校对
3. 热词辅助 LLM：额外提供辩论主题热词表/术语库
```

`src/correct.py` 的提示词要求模型只修明显识别错误；如果整段像字幕、综艺节目名或与辩论无关的幻觉，就输出 `[无法识别]`；同时明确要求不脑补、不解释，只输出纠正后的文本。

## 4. 实验结果怎么样

### 4.1 关键数字

完整结果如下，CER 越低越好：

| 案例 | 类型 | raw CER | 纯 LLM CER | 热词辅助 CER | 现象 |
|---|---|---:|---:|---:|---|
| case1 | 术语错，“熬夜”被识别成“好意” | 0.158 | 0.158 | 0.158 | LLM 改成“好事/好的”，但没有恢复“熬夜” |
| case2 | 术语错，“晚睡晚起”被识别成“玩具” | 0.400 | 0.400 | 1.000 | 纯 LLM 保留原样，热词版反而更差 |
| case3 | 近音错，“多了”被识别成“说了” | 0.077 | 0.077 | 0.077 | 两句都通顺，LLM 没有理由改 |
| case4 | 综艺名幻觉 | 2.286 | 0.571 | 0.571 | LLM 正确拒绝，输出 `[无法识别]` |
| case5 | 字幕式幻觉 | 1.579 | 0.842 | 0.842 | LLM 正确拒绝，输出 `[无法识别]` |

从样本均值看，raw 平均 CER 为 0.900，纯 LLM 后为 0.410，热词辅助后为 0.530。这个均值不能简单解释成“LLM 让识别准确率翻倍”，因为 case4 和 case5 的下降来自拒绝幻觉后的文本变短，而不是恢复出真实语音内容。实验真正有意义的结果是：LLM 能判断“这段不可信”，但不能把已经丢失的声学信息重新变出来。

按案例数量看：

- 纯 LLM：2 条明显改善，3 条不变。
- 热词辅助：2 条改善，2 条不变，1 条变差。
- 字词级错误 case1 / case2 / case3 没有被真正修好。
- 幻觉类错误 case4 / case5 都被识别为不可恢复内容。

### 4.2 最重要的现象

第一个现象是：LLM 对“通顺但错误”的 ASR 文本很无力。比如“我说了两个小时”和“我多了两个小时”都能读通；如果没有声学线索，LLM 并不知道“说”应该改成“多”。同理，“好意”“好事”“好的”都可以在语言上成立，但它们都不是原文“熬夜”。

第二个现象是：热词表不是万能修复器。case2 中，原始 ASR 把“晚睡晚起”识别成“玩具”，两者声学和字形距离都很远。热词表虽然提供了“晚睡晚起”，但模型仍无法可靠桥接，反而把输出推向空结果，CER 从 0.400 变成 1.000。

第三个现象是：LLM 最可靠的价值是拒绝明显幻觉。面对“小明星大跟班下次再見”和“点赞 订阅 转发 打赏”这类与辩论内容明显无关的文本，LLM 能输出 `[无法识别]`，避免把幻觉当成真实转写继续写进报告或下游系统。

## 5. 用到了机器学习课程的什么知识点

本次实验可以对应到课程里的多个知识点：

- **后处理与端到端模型边界**：ASR 不是只看最后文本，错误来源可能在声学前端、解码器、语言模型先验或后处理环节。LLM 后处理只能利用文本信息，不能恢复已丢失的声学证据。
- **消融实验**：固定 5 个案例与 CER 口径，只改变“无纠错 / 纯 LLM / 热词辅助”三个条件，比较每个模块是否真的带来收益。
- **错误类型分析**：把错误分成术语错、近音错、综艺幻觉、字幕幻觉，而不是只报告平均 CER。这能解释为什么同一个方法在不同错误类型上表现不同。
- **鲁棒性评估**：评估系统面对低信噪比、降噪伪影和幻觉输出时是否稳定，而不是只看干净语音上的表现。
- **先验知识的双刃剑效应**：热词表相当于给模型加入任务先验。先验在输入接近正确答案时可能有帮助，但在声学错误太远时可能误导模型。
- **指标与语义目标不完全一致**：输出 `[无法识别]` 会降低某些幻觉案例的 CER，但它并没有恢复真实内容。因此报告里要解释 CER 数字背后的语义含义。
- **分布外错误与幻觉检测**：字幕残留、综艺节目名这类输出不是普通替换错误，而是生成式模型在不确定条件下产生的分布外文本。

## 6. 实验结论和后续建议

实验三的结论可以概括为：LLM 后处理适合作为“幻觉过滤器”，不适合作为“声学信息恢复器”。

更具体地说：

1. 对于整段明显无关的幻觉，LLM 能做出有价值的拒绝，避免系统把错误文本继续传播。
2. 对于近音字、术语错和本身通顺的错词，LLM 很难可靠修正，因为它看不到原始声学信息。
3. 热词表只有在 ASR 输出已经接近正确术语时才可能有帮助；当错误距离太远时，热词表可能没有收益，甚至有害。
4. 本实验样本只有 5 条，且刻意选择了严重退化案例，所以结论应定位为定性边界分析，而不是大规模统计结论。

后续建议：

- 报告中不要把实验三写成“LLM 显著降低 CER”，而应写成“LLM 能识别不可恢复幻觉，但不能稳定修复声学丢失造成的字词错误”。
- 如果还有时间，可以补一小组轻度错误样本，测试“ASR 输出已经接近正确术语”时热词表是否转为正收益。
- demo 中可以加入一个开关：显示 raw ASR 与 LLM 过滤后的文本，让观众看到 `[无法识别]` 的价值不是恢复内容，而是避免胡编。
- 总表里可以把实验三作为“后端语义兜底”的边界实验，与实验一、实验二的声学前端实验形成闭环。

## 7. 实验中失败或不顺的地方，以及可提取的知识点

| 不顺之处 | 表现 | 可提取的知识点 |
|---|---|---|
| LLM 没有修好近音错 | “我说了两个小时”没有改回“我多了两个小时” | 文本后处理无法替代声学判别；缺失的信息不能靠语言模型稳定补回 |
| LLM 没有修好术语错 | “好意/玩具”没有恢复成“熬夜/晚睡晚起” | 术语库只有在错误接近术语时才有效；错误距离太远时先验不够 |
| 热词表帮倒忙 | case2 热词辅助 CER 从 0.400 变成 1.000 | 任务先验可能误导模型，尤其在输入证据不足时 |
| CER 数字容易误读 | 幻觉案例输出 `[无法识别]` 后 CER 下降 | 指标下降不等于内容恢复，需要结合错误类型解释 |
| 第三方中转配置不顺 | 官方端点 401，需要显式配置 `base_url` | 实验复现不仅依赖算法，也依赖服务端点、环境变量和 API 兼容性 |
| thinking 输出混入正文 | 中转把 `<thinking>...</thinking>` 放进正文 block，需要清理 | 使用 LLM API 时要验证输出格式，避免推理文本污染实验结果 |
| 关闭 thinking 时输出不稳定 | 首轮 case1 因推理泄漏导致 CER 异常飙高 | 模型调用参数会影响可复现性，实验脚本必须记录关键配置 |

这些失败点很适合写进课程报告，因为它们说明项目不是简单套模型，而是在分析“哪一类错误应该由声学前端解决，哪一类错误可以由文本后端过滤”。

## 8. 课程报告里可以怎么描述

### 中文表述

> 在实验三中，我们进一步考察了 LLM 作为 ASR 后处理模块的边界。我们从前两个实验中选取 5 条代表性错误输出，覆盖术语错误、近音错误、综艺节目名幻觉和字幕式幻觉。实验采用盲测设置：纠错时只向模型提供 ASR 转写文本，不提供参考答案，并比较无纠错、纯 LLM 纠错和热词辅助 LLM 纠错三种设置。

> 结果表明，LLM 后处理并不能稳定修复字词级 ASR 错误。对于“我说了两个小时/我多了两个小时”这类近音错误，或者“好意/熬夜”这类通顺但错误的术语替换，LLM 缺少原始声学线索，因此通常会保留错误或改成另一个同样通顺但仍然错误的表达。热词表也没有稳定收益；当 ASR 输出与正确术语距离过远时，热词先验无法桥接声学错误，甚至可能使结果变差。

> 然而，LLM 在识别明显幻觉方面具有实际价值。对于“小明星大跟班下次再见”或“点赞、订阅、转发”等明显与辩论无关的输出，LLM 能够拒绝脑补并输出 `[无法识别]`。因此，实验三的主要结论是：LLM 更适合作为 ASR 系统的幻觉过滤器，而不是声学错误恢复器。它可以减少错误文本继续传播，但不能替代前端降噪、分离或更可靠的声学识别。

### English wording

> In Experiment 3, we examined the boundary of using an LLM as a post-processing module for ASR. We selected five representative ASR errors from the previous experiments, including terminology errors, near-homophone errors, TV-show hallucinations, and subtitle-like hallucinations. The correction was performed in a blind setting: the model only received the ASR transcript, without access to the reference text. We compared three settings: no correction, pure LLM correction, and LLM correction with a task-specific hotword list.

> The results show that LLM post-processing cannot reliably recover word-level ASR errors once the acoustic evidence has been lost. For example, when "duo" was recognized as "shuo", or when a debate-related term was replaced by another fluent phrase, the LLM had no acoustic clue to decide which word was correct. The hotword list also did not provide consistent gains. When the ASR output was too far from the intended term, the hotword prior could not bridge the acoustic mismatch and even made one case worse.

> However, the LLM was useful for detecting obvious hallucinations. When the ASR output contained unrelated TV-show names or subtitle-like phrases such as "like, subscribe, and donate", the LLM correctly rejected the content by outputting `[unrecognizable]` instead of inventing a plausible debate sentence. Therefore, the key lesson is that LLMs are better used as hallucination filters than as acoustic error recoverers in this setting.

## 9. 本次总结的一句话结论

实验三把项目主线补完整了：前端预处理解决的是“声音还在不在、能不能听清”的问题，而 LLM 后处理解决的是“文本看起来是否可信”的问题；一旦声学信息已经丢失，LLM 最可靠的做法不是猜答案，而是承认无法识别。
