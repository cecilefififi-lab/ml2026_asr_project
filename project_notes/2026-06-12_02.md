## 2. 本次实验在做什么

本次实验的核心问题是：音频前端降噪是否真的能提升 ASR 鲁棒性？

实验比较了三种输入条件：

- 不做降噪，直接送入 ASR。
- 使用 FRCRN_SE_16K 做神经网络降噪。
- 使用 `noisereduce` 的 spectral gating 做谱减法降噪。

每种条件都在同一批 26 条音频上评测，并覆盖：

- 2 类噪声：white noise 和 babble noise。
- 3 档 SNR：15 dB、5 dB、0 dB。
- 2 个 ASR 引擎：Whisper large-v3 和 FunASR paraformer-zh。
- clean 音频也过降噪，用来验证“高质量音频上降噪可能引入伪影”的假设。

原计划尝试 DeepFilterNet，但 Python 3.12 下缺少可直接使用的 `deepfilterlib` wheel，源码安装又需要 Rust 工具链。为了不扩大依赖复杂度，实验改用已在环境中可跑通的 ClearVoice FRCRN_SE_16K，作为神经降噪代表；谱减法作为简单 DSP 代表。

## 4. 实验流程

本次主流程可以概括为：

```text
clean speech
  -> 加 white / babble 噪声
  -> 得到 15 dB、5 dB、0 dB noisy speech
  -> 分别执行：不处理 / FRCRN / 谱减法
  -> Whisper 与 FunASR 识别
  -> 计算 CER、ASR RTF、降噪 RTF
  -> 汇总表格、图和典型案例
```

其中 clean 音频也进入 FRCRN 和谱减法，是本次实验设计里很重要的一点。它不是为了模拟真实噪声，而是为了检查：如果音频本来已经较干净，额外降噪会不会因为频谱伪影反而让 ASR 更差。

## 5. 实验结果怎么样

### 5.1 关键数字

下表是平均 CER，数值为“无处理 -> FRCRN -> 谱减法”。

| 条件 | Whisper CER | FunASR CER |
|---|---:|---:|
| clean | 4.9% -> 9.3% -> 14.7% | 10.1% -> 8.5% -> 13.0% |
| white 15 dB | 7.4% -> 16.6% -> 14.4% | 9.6% -> 12.0% -> 14.6% |
| white 5 dB | 12.2% -> 33.2% -> 32.3% | 29.4% -> 17.0% -> 16.8% |
| white 0 dB | 41.9% -> 49.9% -> 61.7% | 68.4% -> 27.5% -> 40.1% |
| babble 15 dB | 5.4% -> 17.0% -> 19.0% | 8.5% -> 12.7% -> 16.1% |
| babble 5 dB | 20.6% -> 78.6% -> 47.1% | 22.3% -> 52.3% -> 30.1% |
| babble 0 dB | 71.7% -> 92.5% -> 83.7% | 43.3% -> 95.1% -> 46.2% |

降噪耗时方面，谱减法平均 RTF 约 0.018，FRCRN 平均 RTF 约 0.105。也就是说，FRCRN 的前端耗时大约是谱减法的 6 倍。链路总 RTF 需要把降噪 RTF 和 ASR RTF 加起来看。

### 5.2 图表现象

`results/exp1_denoise_curves.png` 可以作为报告里的主图。它展示了同一噪声类型下，CER 随 SNR 降低而上升，同时能直观看出降噪曲线并不总是在无处理曲线下方。

`results/exp1_denoise_heatmap.png` 更适合放在展示或视频里。它把“模型、噪声、SNR、降噪方法”的组合压缩成颜色，最醒目的格子是 FunASR 在 white 0 dB 下被 FRCRN 明显救回，以及 FunASR 在 babble 0 dB 下被 FRCRN 严重伤害。

### 5.3 主要发现

1. 降噪不是稳定增益。它只在少数组合下明显有效，尤其是 FunASR 面对 white noise 的低 SNR 场景。
2. FunASR 在 white 0 dB 下 CER 从 68.4% 降到 27.5%，这是本次最明确的“降噪有用”结果。
3. Whisper 几乎没有从降噪中获益。多数条件下降噪反而提高 CER，说明 Whisper 自带的鲁棒性在这些短音频上已经强于前端降噪带来的净收益。
4. babble noise 下神经降噪尤其危险。FunASR 在 babble 0 dB 下从 43.3% 恶化到 95.1%，说明 FRCRN 可能把目标中文语音也当成背景人声消掉，或者留下更容易误导 ASR 的残留语音。
5. clean 和 15 dB 条件下降噪经常有害，验证了“高 SNR 场景中前端伪影可能伤害 ASR”的假设。
6. “神经降噪一定优于传统方法”的假设不成立。FRCRN 在 white noise 上更强，但在 babble noise 下比谱减法更容易造成灾难性错误。
7. 降噪会改变错误类型。Whisper 在低 SNR 下可能从“整句幻觉”变成“空输出”，这对后续 VAD 和幻觉实验很有价值。

## 6. 典型案例

`results/exp1_cases.txt` 中已经筛出 5 个代表案例，可以直接用于报告或视频。

| 类型 | 条件 | 现象 |
|---|---|---|
| 降噪帮忙 | `white_5dB / funasr / specsub / pro_015.wav` | 无处理输出为空，谱减法后正确识别“我多了两个小时”，CER 从 100% 到 0% |
| 降噪帮忙 | `white_0dB / funasr / frcrn / pro_011.wav` | 无处理输出为空，FRCRN 后识别出主体句子，CER 从 100% 到约 10% |
| 压制幻觉 | `white_0dB / whisper / con_009.wav` | 无处理出现“小明星大跟班下次再见”式幻觉，降噪后幻觉消失，但文本仍不完全正确 |
| 降噪帮倒忙 | `babble_0dB / funasr / frcrn / con_001.wav` | 原本还能识别部分中文，FRCRN 后输出英文句子，说明目标语音可能被错删，背景人声残留主导了 ASR |
| 诱发新幻觉 | `white_0dB / whisper / specsub / pro_002.wav` | 谱减法后出现“请不吝点赞 订阅 转发 打赏支持...”这类字幕残留式幻觉 |
| 消灭正确答案 | `babble_0dB / whisper / specsub / con_007.wav` | 无处理 CER 为 0%，降噪后输出为空 |

这些案例比单纯的平均数更能说明“为什么预处理不能盲目套用”。平均 CER 告诉我们结果变好还是变差，案例告诉我们错误是怎么发生的。

## 7. 用到的机器学习课程知识点

本次实验和机器学习课程知识的连接很清楚：

- **消融实验**：固定数据、噪声和 ASR 模型，只改变前端处理方法，观察 FRCRN 和谱减法是否带来增益。
- **鲁棒性评测**：用 SNR 控制扰动强度，观察模型在分布偏移下的性能退化。
- **模型比较**：Whisper 和 FunASR 的错误模式不同，说明不能用单一模型结论概括所有 ASR。
- **评价指标设计**：CER 衡量准确率，RTF 衡量效率，二者共同构成 accuracy-efficiency trade-off。
- **数据增强与受控变量**：white / babble 噪声和固定 SNR 让实验可复现，也让不同条件之间可比较。
- **预训练模型的域适配问题**：FRCRN 在稳态 white noise 上有效，但在 babble 人声背景下可能不适配。
- **错误分析**：不仅看平均 CER，还分析空输出、幻觉、语言切换和目标语音被错删等失败模式。
- **偏差与泛化**：同一降噪模型在一种噪声上有效，在另一种噪声上失败，说明模型性能依赖训练分布和测试场景匹配程度。

## 8. 实验结论和后续建议

本次实验的核心结论是：降噪不应该被当成 ASR 前面的默认步骤。它是一个有条件收益的模块。

更具体地说：

1. 如果噪声是 white noise 且 SNR 很低，尤其是 FunASR 这类对宽带能量掩蔽敏感的系统，降噪有明显价值。
2. 如果噪声是 babble noise，神经降噪可能把背景人声、目标人声和伪影混在一起，反而显著伤害 ASR。
3. 如果音频本来已经比较干净，额外降噪往往不划算，因为伪影代价可能大于去噪收益。
4. Whisper 对前端降噪的收益很有限，后续如果时间紧，可以优先把降噪链路用于 FunASR 或特定噪声条件，而不是全量套用。
5. 后续实验二在“噪声 + 重叠语音”场景下，不应默认先降噪再分离，而应该同时比较“先降噪”和“先分离”。

后续建议：

- 实验二优先保留 white 0 dB / 5 dB 和 babble 0 dB 这几个最能拉开差异的条件。
- 对 Whisper 幻觉单独做小实验，把 `pro_002` 和 `con_009` 这类案例作为视频高光素材。
- 在报告中把 FRCRN 描述为“条件有效”的前端，而不是“更高级所以更好”的方法。
- 如果需要做 demo，不要默认展示 babble + FRCRN 的链路；它更适合作为失败案例。
- 在消融总表里把“平均 CER”和“典型失败模式”放在一起，避免只用均值掩盖灾难性个例。

## 9. 实验中失败或不顺的地方

| 问题 | 表现 | 可以提取的知识点 |
|---|---|---|
| DeepFilterNet 没有继续使用 | Python 3.12 下缺少可用 wheel，源码安装需要额外 Rust 工具链 | 工程约束会影响模型选择；课程项目要优先保证可复现和按时完成 |
| FRCRN 在 babble 下严重失败 | FunASR babble 0 dB 从 43.3% 恶化到 95.1% | 神经网络前端存在域不匹配问题；背景人声不是普通噪声 |
| clean 音频降噪后变差 | Whisper clean 从 4.9% 恶化到 9.3% / 14.7% | 预处理会引入伪影；高 SNR 场景不一定需要增强 |
| Whisper 出现训练数据残留式幻觉 | 谱减后出现“点赞、订阅、转发”式输出 | 生成式 ASR 可能在低置信音频下由语言模型先验接管 |
| FunASR 批跑时中途卡住 | 某些目录批处理在模型初始化阶段长时间无进展 | 长批量实验应支持断点续跑、分条件运行和缺失补跑 |
| Windows 中文输出编码容易错位 | 控制台中文输出需要 UTF-8 设置 | 实验日志和结果文件应统一编码，避免报告材料损坏 |

这些问题可以在报告里归入 failure analysis 或 limitations。它们不是单纯的错误，而是解释“为什么实际 ASR pipeline 不能只堆模型”的证据。

## 10. 课程报告里可以怎么描述

### 中文表述

> 在实验一中，我们研究了本地降噪前端对 ASR 鲁棒性的影响。实验使用 26 条人工校对的中文辩论短音频，分别叠加 white noise 和 babble noise，并设置 15 dB、5 dB、0 dB 三档信噪比。我们比较了三种输入条件：不做降噪、FRCRN 神经降噪和谱减法降噪，并在 Whisper large-v3 与 FunASR paraformer-zh 上计算字符错误率和实时率。

> 结果表明，降噪并不是稳定有效的预处理。它在 FunASR 面对低信噪比 white noise 时效果明显，例如 white 0 dB 下 CER 从 68.4% 降至 27.5%。但在 babble noise 下，神经降噪可能严重破坏目标语音，FunASR 在 babble 0 dB 下的 CER 从 43.3% 上升到 95.1%。对于 Whisper，降噪在大多数条件下没有带来收益，反而常常因为伪影或空输出导致错误率上升。因此，本实验的主要结论是：音频增强不应作为默认步骤，而应根据噪声类型、信噪比和 ASR 模型特性有条件地使用。

> 一个重要失败案例是，在 babble 0 dB 条件下，FRCRN 会把中文目标语音与英文背景人声混淆，导致 FunASR 输出英文句子。这说明背景人声噪声和稳态噪声不同，神经降噪模型如果训练分布不匹配，可能比不处理更糟。

### English wording

> In Experiment 1, we evaluated whether local denoising front-ends actually improve ASR robustness. We used 26 manually corrected Chinese debate clips and mixed them with two types of noise, white noise and babble noise, at three SNR levels: 15 dB, 5 dB, and 0 dB. For each noisy condition, we compared three front-end settings: no denoising, FRCRN neural speech enhancement, and spectral-gating-based noise reduction. We then evaluated Whisper large-v3 and FunASR paraformer-zh using character error rate (CER) and real-time factor (RTF).

> The results show that denoising is not a universally beneficial preprocessing step. It helped FunASR substantially under low-SNR white noise; for example, under white noise at 0 dB, the CER dropped from 68.4% to 27.5% with FRCRN. However, the same neural denoiser was harmful under babble noise. For FunASR under babble noise at 0 dB, the CER increased from 43.3% to 95.1%, suggesting that the denoiser may remove parts of the target speech or leave misleading speech-like artifacts. For Whisper, denoising did not improve performance in most conditions and often increased CER due to artifacts or empty transcriptions.

> These findings suggest that audio enhancement should not be treated as a default ASR preprocessing step. Its benefit depends on the noise type, SNR level, and ASR backend. In particular, human-speech-like background noise is much harder than stationary white noise, and a stronger neural front-end is not necessarily safer than a simpler DSP method.

## 11. 本次总结的一句话结论

实验一把项目主题从“噪声会让 ASR 变差”推进到了更有价值的问题：“什么噪声、什么模型、什么前端组合下，预处理才真正值得做？”答案不是简单的 yes or no，而是一个条件性结论：降噪有用，但只在合适的噪声类型、合适的 SNR 和合适的 ASR 后端上有用。
