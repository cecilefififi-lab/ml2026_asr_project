## 2. 本次实验在做什么

本次主要是在建立一个最小可运行的 ASR 鲁棒性评测框架。核心问题是：当语音受到不同类型、不同强度的噪声干扰时，ASR 模型的识别准确率会如何变化？

实验没有直接进入复杂降噪或 LLM 修正，而是先完成项目地基：

- 准备一批可以人工校对的中文短音频。
- 为每条音频准备 ground truth 文本。
- 合成两类噪声：white noise 和 babble noise。
- 按 SNR 15 dB、5 dB、0 dB 生成加噪音频。
- 用 Whisper 和 FunASR 分别识别 clean / noisy 音频。
- 用 CER 评价识别错误率，用 RTF 评价处理效率。
- 初步验证语音分离模型是否可用于后续重叠语音实验。

简单来说，本次是在回答一个基础问题：不做任何降噪时，噪声本身会怎样破坏 ASR？

## 4. 实验流程

本阶段的主要流程如下：

```text
干净音频
  -> 人工校对参考文本
  -> 生成 white / babble 噪声
  -> 按 15 dB、5 dB、0 dB 叠加噪声
  -> Whisper / FunASR 识别
  -> 计算 CER 和 RTF
  -> 汇总表格与退化曲线
```

语音分离部分的流程是：

```text
重叠语音样本
  -> SepFormer / MossFormer2 分离测试
  -> ASR 转写或波形相关性检查
  -> 判断分离是否真的成功
  -> 决定后续实验使用自制可控重叠样本
```

## 5. 实验结果

人工校对参考文本后，阶段一得到的主要 ASR 结果如下：

| 条件 | Whisper CER | FunASR CER |
|---|---:|---:|
| clean | 5.3% | 10.4% |
| white 15 dB | 7.8% | 10.0% |
| white 5 dB | 12.5% | 29.8% |
| white 0 dB | 42.1% | 68.4% |
| babble 15 dB | 5.7% | 8.8% |
| babble 5 dB | 20.9% | 22.5% |
| babble 0 dB | 71.9% | 43.6% |

关键观察：

- Whisper 在 clean 条件下表现更好，CER 约 5.3%。
- FunASR 在 clean 条件下 CER 约 10.4%，但速度更快。
- white noise 0 dB 对 FunASR 伤害很大，CER 达到 68.4%。
- babble noise 0 dB 对 Whisper 伤害更大，CER 达到 71.9%，并出现明显幻觉式输出。
- 两个模型的弱点不完全一样，说明 ASR 鲁棒性不能只用一个模型或一种噪声判断。
- RTF 大体稳定：Whisper 约 0.26-0.31，FunASR 约 0.08-0.15，噪声强度对运行速度影响不明显。

语音分离验证结果：

- SepFormer 在真实 `MidOverlap.wav` 上基本失败，两路输出内容高度相似。
- MossFormer2 在真实 `MidOverlap.wav` 上也没有稳定分开，输出仍接近混合语音。
- MossFormer2 在自制完全重叠样本上能工作，分离输出与两个源语音相关性约为 0.82 / 0.79。
- 因此后续重叠实验决定改用自制可控重叠样本，而不是完全依赖学长已有的真实重叠录音。

## 6. 用到的机器学习课程知识点

本阶段虽然还没有训练新模型，但已经用到了很多机器学习实验设计知识：

- **Baseline 思维**：先建立最小可运行基线，再比较后续降噪、分离、LLM 修正是否真的有增益。
- **监督评测**：使用人工校对文本作为 ground truth，用 CER 计算模型输出和参考答案的差异。
- **评价指标**：用 CER 衡量准确率，用 RTF 衡量效率，体现 accuracy-efficiency trade-off。
- **数据增强 / 扰动测试**：通过添加不同类型和强度的噪声，模拟真实环境中的分布变化。
- **鲁棒性分析**：观察模型在 clean 到 noisy 条件下性能退化的规律。
- **模型比较**：比较 Whisper 和 FunASR 两类 ASR 系统在不同噪声下的行为差异。
- **信噪比 SNR**：用 15 dB、5 dB、0 dB 控制噪声强度，形成可复现实验条件。
- **神经网络模型应用**：Whisper、FunASR、SepFormer、MossFormer2 都是深度学习语音模型，本阶段主要使用预训练模型进行推理和对比。
- **泛化与失败案例分析**：模型在自制重叠样本上可分离，但在真实重叠录音上失败，说明模型能力受数据分布影响。

## 7. 实验结论与后续建议

本阶段的主要结论是：

1. 噪声确实会显著破坏 ASR，尤其在 0 dB 低信噪比条件下。
2. 不同 ASR 模型对噪声类型的敏感性不同，Whisper 更容易被 babble 人声噪声诱发幻觉，FunASR 对 white noise 的退化更明显。
3. 只看 clean 准确率不够，模型在真实噪声下的鲁棒性可能完全不同。
4. 语音分离模型并不是即插即用的，真实重叠语音比合成重叠样本更难。
5. 后续实验应该围绕“降噪是否真的改善 ASR”和“噪声 + 重叠时先降噪还是先分离”继续推进。

后续建议：

- 在实验 1 中加入至少一种神经降噪方法和一种简单 DSP 方法。
- 保留 clean / noisy baseline 作为所有后续实验的对照组。
- 重叠语音实验优先使用自制可控样本，这样可以明确知道每个说话人的参考文本。
- 学长真实重叠样本可以作为定性失败案例，而不是主定量评测数据。
- 报告中不要只说“用了很多模型”，而要强调不同模型在不同噪声下的失败模式不同。

## 8. 失败、问题与可提取的知识点

本阶段有几个重要失败或调整：

| 问题 | 表现 | 可以提取的知识点 |
|---|---|---|
| AliMeeting 数据没有继续使用 | 数据量较大，且对当前小型鲁棒性实验不必要 | 课程项目要控制数据规模，优先保证实验可解释、可完成 |
| MUSAN 没有作为主要噪声源 | 改用 white 和 babble 两类可控噪声 | 可控变量比追求复杂数据集更重要 |
| Whisper 草稿不能直接当参考文本 | clean/Whisper CER 会被构造性压低 | ground truth 不能直接来自被测模型，必须人工校对 |
| SepFormer 分离真实重叠语音失败 | 两路输出内容几乎相同 | 预训练模型存在数据域不匹配问题 |
| MossFormer2 分离真实 `MidOverlap.wav` 也不稳定 | 输出与混合语音高度相关 | 模型在论文或他人流程中成功，不代表换环境后必然复现 |
| 短音频可能导致 ASR 不稳定 | 当前片段多为 2-4 秒 | 后续如果出现不稳定，可以考虑拼接或 context padding |

这些失败不是坏事，反而是报告里的亮点：它们说明项目不是只跑通一个 pipeline，而是在真实地分析模型什么时候可靠、什么时候不可靠。

## 9. 课程报告里可以怎么说

可以在报告中这样描述阶段一：

> We first built a minimal ASR robustness evaluation pipeline. The clean speech set contains 26 manually corrected Chinese debate clips. For each clip, we generated noisy versions by adding white noise and babble noise under three SNR levels: 15 dB, 5 dB, and 0 dB. We then evaluated two ASR systems, Whisper and FunASR, using character error rate (CER) and real-time factor (RTF).

中文报告可以这样写：

> 本项目首先构建了一个最小 ASR 鲁棒性评测基线。我们使用 26 条人工校对的中文辩论短音频作为 clean speech，并分别叠加 white noise 和 babble noise，在 15 dB、5 dB、0 dB 三档信噪比下比较 Whisper 与 FunASR 的字符错误率和实时率。实验发现，不同 ASR 模型对噪声类型的敏感性并不一致：Whisper 在 babble 0 dB 下出现明显退化和幻觉式输出，而 FunASR 对 white noise 的退化更加明显。这说明后续预处理策略不能只基于单一模型或单一噪声假设。

关于失败案例可以这样写：

> 在语音分离验证中，我们发现 SepFormer 和 MossFormer2 对真实 `MidOverlap.wav` 的分离效果都不稳定，两路输出仍高度接近混合语音。但在自制可控重叠样本上，MossFormer2 能够分离出与源语音相关性较高的两路音频。因此，后续定量实验改用自制重叠样本，以保证 ground truth 清晰可控；真实重叠录音则作为定性失败案例分析。

这部分可以放在报告的实验设置、baseline 结果和 failure analysis 中。
