# 视频脚本大纲（中文规划稿）

> **项目**：噪声与重叠语音场景下，本地音频预处理对 ASR 鲁棒性的影响研究
> **英文片名**：*Who Said That?! Teaching ASR to Survive a Debate*
> **目标时长**：≥10 分钟（本大纲按 ~12 分钟排布，可裁剪到 10 分钟）
> **成片语言**：英文录制（课程要求）。本大纲为中文规划稿，旁白为中文要点，录制时翻成英文；技术术语（Whisper / FunASR / CER / VAD / SNR 等）保持英文。
> **叙事主线（一句话）**：辩论赛书记员的难题——观众起哄（噪声）+ 选手抢话（重叠），AI 到底在哪里听错，哪些预处理真有用、哪些看起来高级却不划算。
> **贯穿隐喻**：噪声 = 观众起哄；重叠 = 抢话；LLM 纠错 = 书记员事后校对；VAD = "现在到底有没有人在说话"的门卫。
> **评分维度对照**（老师明确：insight / nuances / depth / difficulties / **fun**，看迭代过程不看代码量）：每段末标注本段主打哪一维。

## 段 5 — 实验一：噪声 × 降噪 × ASR（4:30–6:00）

- **画面**：先放退化曲线 `results/degradation_curve.png`，再放降噪曲线/热力图 `results/exp1_denoise_curves.png` / `exp1_denoise_heatmap.png`。
- **旁白要点**：
  1. **噪声退化**：SNR 15→5→0 dB，CER 单调上升。**两模型弱点相反**——Whisper 抗 white 强（5dB 仅 12.5%）但 babble 0dB 崩到 71.9%（大量整句幻觉）；FunASR 怕 white（0dB 68.4%）却在 babble 0dB 更稳（43.6%）。两条曲线在 0dB 处交叉。
  2. **机理解释**（nuance）：Whisper 是生成式/多语言，可懂的人声型噪声（babble）诱发其语言模型"接管"产生幻觉；FunASR 是判别式 CTC，对宽带能量掩蔽（white）更敏感。
  3. **降噪不是万能开关**：全 6 噪声条件里，降噪只在**一种组合**大幅有效——FunASR × white × 0dB（68.4→27.5）；Whisper 任何场景都不从降噪受益；clean/15dB 下降噪反而有害（Whisper clean 4.9→9.3/14.7，伪影假设证实）。
  4. **反直觉灾难**：babble + 神经降噪 = 灾难，FunASR babble_0dB 43.3→**95.1**——FRCRN 分不清目标人声和背景人声，把目标语音消掉了。
- **典型案例**（可放前后文本对比卡片）：
  - 帮倒忙①"删错了人"：FRCRN 把中文目标消掉、留下背景英文 babble。
  - 帮倒忙②"诱发新幻觉"：specsub 后冒出"請不吝点赞订阅转发打赏…"（实验四伏笔）。
  - 帮忙①"救活 FunASR"：white_5dB 空输出 → specsub 后完美转写（CER 100→0）。
- **数据/出处**：LOG.md 2026-06-11 退化矩阵、2026-06-12 实验一表格与案例；`results/exp1_cases.txt`
- **维度**：nuances（"降噪有时救命、有时帮倒忙"的条件边界）

## 段 6 — 实验二：噪声 + 重叠语音处理顺序（6:00–7:30）

- **画面**：五链路对比图 `results/ablation_pipelines.png` + 透视表 `results/exp2_pivot.md`（高亮每行最优）。配案例音频前后试听。
- **旁白要点**：
  1. **直接 ASR（L1）几乎总赢**：clean/no L1=4.4% vs 任何分离链路 47–70%；L1 在 15 个条件里 **13 个最优**，分离 L3 仅 2 个最优（且都是 heavy 重叠）。
  2. **分离基本没成功**：per-speaker CER 全程 63–112%，典型失败模式是把混合**复制到两路**（spk1≈spk2≈整句混合）——content CER 看着还行，正是因为两路都召回了全部内容、却都没分开。
  3. **处理顺序之谜被推翻**：原假设"先降噪再分离（L4）更稳"**不成立**，L4/L5 差异小、无稳定赢家。
  4. **何时分离才回本**：只有两人长时间同步抢话（heavy）时分离才勉强反超（clean/heavy 46.9<50.0；babble_0dB/heavy 70.3<90.7）。
  5. **工程代价**：分离链路耗时约为 L1 的 2–3 倍，换来的却是更差的 CER —— trade-off 全程负向。
- **典型案例**：案例 A（轻度重叠分离复制失败）；案例 B（babble_0dB 两路皆字幕幻觉"中文字幕志愿者""感谢观看"）。
- **数据/出处**：LOG.md 2026-06-13 全表与假设回顾；`results/exp2_pivot.md`、`exp2_summary.csv`
- **维度**：depth（按重叠度×SNR×链路系统扫描，假设被数据推翻并解释）

## 段 7 — 实验三：LLM 语义纠错的边界（7:30–8:30）

- **画面**：5 案例三档对比表 `results/exp3_correction.md`（raw / 纯 LLM / 热词辅助）。
- **旁白要点**：
  1. 三档盲测（纠错时不给参考；模型 claude-opus-4-8，要求"整段是幻觉就输出 `[无法识别]`、不准脑补"）。
  2. **LLM 唯一可靠价值 = 识别并拒绝幻觉**（case4 综艺名、case5 字幕残留 → 正确判 `[无法识别]`，不编造）。
  3. **声学信息已丢失的字词错，LLM 修不了**（case1/2/3）：错词本身语法语义通顺（"好意/说了/玩具"都读得通），LLM 没有声学线索，要么保留、要么换成另一个通顺的错词。
  4. **热词表无增益甚至帮倒忙**（case2 纯 LLM 0.40 → 热词 1.0）：当 ASR 输出离术语太远（"玩具" vs "晚睡晚起"），热词无法桥接，反而诱导误判。
  5. **诚实 caveat**：样本仅 5 条且偏向严重退化案例，结论为定性；轻度错误下 LLM/热词可能转正收益——本实验刻意展示的是 LLM 兜底的**边界**。
- **数据/出处**：LOG.md 2026-06-14 表格与假设回顾；`results/exp3_correction.md` / `.csv`
- **维度**：nuances（区分"语言层可修" vs "声学层已丢失"）

## 段 8 — 实验四：Whisper 幻觉小实验（fun 段）（8:30–9:30）

- **画面**：VAD 对比图 `results/exp4_vad_compare.png` + 幻觉案例集 `results/exp4_hallucination.md`（字幕式输出截图）。
- **旁白要点**：
  1. 给 Whisper 喂**纯噪声 / 静音 / 甚至纯零采样**，它不沉默，而是吐训练数据残留型幻觉："由 Amara.org 社群提供的字幕""请不吝点赞订阅打赏""字幕志愿者 杨茜茜"。VAD off 下 91% 命中。
  2. **反直觉**：纯零静音（全 0）照样稳定吐"Amara.org"——Whisper 不靠能量门控决定是否解码。
  3. **VAD 是双刃 / 有盲区**：white + 静音 100%→0%（完全压制）；**babble 67%→67%（几乎无效）**——Silero VAD 把可懂人声型噪声当成语音放行。**这正解释了实验一为何 babble 0dB 最毒：它骗得过 VAD。** 辩论现场观众嘈杂本质就是 babble。
  4. **跨实验复现**：同款字幕幻觉在实验一/二/四反复出现，证明是 Whisper 固定模式而非随机；机理 = 声学证据不足时语言模型先验接管，从训练记忆调出"无语音段字幕"填空。
- **数据/出处**：LOG.md 2026-06-15 实验四表 1/表 2 与假设回顾；`results/exp4_hallucination.md`
- **维度**：fun（"AI 在没人说话时自言自语打广告"既好笑又有 insight，并回收段 4 伏笔）

## 段 9 — 真实录音泛化抽查 + Demo（9:30–11:00）

- **画面**：先并排表 `results/real_spotcheck.md`，再切到 Streamlit demo（`demo/app.py`）现场演示。
- **旁白要点（泛化抽查）**：
  - 换 6 条手机实录（宿舍/食堂/教室/三种双人讨论），不强行算 CER，做人工 spot-check（文稿允许意译、重叠无逐字 GT —— 诚实说明为何不造精确数字）。
  - **真实录音独立复现全部主结论**：直接 ASR 最稳；canteen+谱减末尾冒"请不吝点赞订阅订阅"（降噪反害）；分离只在 heavy 同步重叠回本（spk2 找回直接 ASR 丢掉的 A 的同步句）；L4 降噪→分离吐"中文字幕志愿者 李宗盛"（链式放大幻觉）；VAD 双刃（classroom 消尾部幻觉=益、canteen 切真实语音=害）。
  - **nuance**："噪声+自然插话"反而比"安静重叠"转得更完整——伤害 ASR 的是**同步重叠时长**，不是有没有噪声。

  - Tab2 可上传一条音频现场跑直接 ASR，展示文本+耗时+RTF。

## 段 10 — 结论、局限与分工（11:00–12:00）

- **画面**：精度-耗时 trade-off 散点 `results/tradeoff.png`（没有链路比 L1 又快又准）+ 消融主表 `results/ablation_summary.md` 浓缩成一张"help / hurt / neutral"决策卡 + 团队署名。
- **旁白要点（结论）**：
  - 一句话主结论：**2–4 秒短辩论片段 + 噪声/重叠约束下，直接 ASR 几乎总是性价比最高；复杂前端只在极窄条件回本**（FunASR×white×低 SNR 的降噪、heavy 重叠的分离），复杂度每加一层多数条件 CER 不降反升。
  - 规则版"什么时候用什么"：单人干净/轻噪→直接 ASR（必要时 VAD 去尾部幻觉）；单人强人声噪声→别默认降噪；自然插话双人→直接 ASR；重度同步重叠→可试分离但需人工核对两路是否互补；降噪→分离链路慎用；热词只在声学接近时有效。
- **局限（limitations，诚实收尾）**：
  - 片段短（2–4s）、样本规模小、单一辩论语料；真实录音是 spot-check 而非大样本定量；MossFormer2 在本数据短片段分离普遍失败（学长同模型能分开，疑有未公开预处理）；实验三仅 5 条偏严重退化案例。
  - future work：context padding 缓解短片段分离、扩展真实场景、规则版链路选择器。
- **遇到的困难（difficulties，可快剪一屏踩坑墙）**：
  - DeepFilterNet 无 Python 3.12 wheel → 改 FRCRN；
  - MossFormer2 环境一波三折（Windows symlink 需 COPY 策略、Python 3.12 speechbrain LazyModule 崩溃、pyarrow 必须先于 librosa 导入避免 DLL 段错误）；
  - 6GB 显存装不下分离+降噪+ASR 三模型 → 按环节分进程、中间产物落盘串联；
  - 本机无 ffmpeg → 用 venv 自带 PyAV 内置 ffmpeg 解码 m4a；
  - 第三方中转把 thinking 塞进正文 → 正则 strip 标签；
  - Windows GBK 控制台遇中文/幻觉文本崩溃 → 全程 `PYTHONIOENCODING=utf-8`。
- **分工与 AI 协作**（片尾署名）：
  - hzy：噪声分析脚本与 CSV、`correct.py`、exp3 输入；tjx：实验一数据处理与 exp3 结果整理；jxy：音频数据清洗、每日记录与 session note；hsf、gxy：实验二所需音频录制。
  - AI 协作者（如 codex / Claude Code）在提交记录中以 Co-author 形式体现，直观呈现真实协作过程。
- **数据/出处**：`results/ablation_summary.md`；project_notes/SESSION_NOTES.md（分工）；各实验 LOG 踩坑段
- **维度**：insight + difficulties（回答总问题 + 真实坦白局限和踩坑）
